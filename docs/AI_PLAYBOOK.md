## Как работать с проектом

### Добавить новый Blueprint

1. Создать папку `src/content/blueprints/[slug]/`
2. Создать `meta.json`:
```json
{
  "title": "Сайт компании",
  "description": "Корпоративный сайт с блогом и формами",
  "difficulty": "easy",
  "phases": 4,
  "totalCards": 12,
  "totalXP": 250
}
```
3. Создать файлы фаз: `01-structure.mdx`, `02-technology.mdx`...
4. Каждая карточка в MDX:
```mdx
---
cardKey: "design-tokens"
title: "Создать Design Tokens"
xpReward: 20
time: "30 минут"
aiPrompt: "Объясни что такое дизайн-токены и как их создать"
decision: "Какие токены нужны именно этому проекту?"
---

## Design Tokens

Дизайн-токены — это переменные для цветов, шрифтов и отступов...

### Альтернативы
- CSS-переменные (рекомендуем)
- Tailwind CSS конфиг
- SCSS переменные

### 🤖 Спросить AI
Если непонятно зачем это нужно — спроси AI.
```

### Обновить карточку

1. Найти MDX файл
2. Обновить содержимое
3. Если меняется `xpReward` — обновить `meta.json` (totalXP)

### Добавить новую фичу

1. Проверить по MANIFEST.md — не нарушает ли принципы?
2. Создать ADR в `docs/DECISIONS/`
3. Реализовать
4. Обновить ARCHITECTURE.md и AI_PLAYBOOK.md

## Правила для AI

### ✅ Что можно делать самостоятельно

- Обновлять содержимое карточек (MDX)
- Создавать новые Blueprint'ы по шаблону
- Предлагать улучшения UX
- Объяснять пользователю решения (через AI-консультанта)

### ❌ Что НЕ делать без спроса

- Менять MANIFEST.md
- Менять архитектуру базы данных
- Добавлять новые зависимости без ADR
- Усложнять интерфейс

### Когда спрашивать пользователя

- Выбор между двумя равнозначными технологиями
- Изменение UX-паттерна
- Всё что влияет на «новичок поймёт?»

## Структура проекта (для AI)

```
proektmap.ru/
├── docs/                  ← Документация (читать первой!)
│   ├── MANIFEST.md        ← Конституция
│   ├── ARCHITECTURE.md    ← Архитектура
│   ├── AI_PLAYBOOK.md     ← Этот файл
│   ├── DECISIONS/         ← ADR'ы
│   └── BLUEPRINTS/        ← Шаблоны Blueprint'ов
├── src/
│   ├── app/               ← Next.js App Router
│   ├── content/blueprints/← MDX файлы Blueprint'ов
│   ├── lib/
│   │   ├── db/            ← Prisma клиент
│   │   ├── game/          ← XP-система
│   │   └── ai/            ← OpenRouter клиент
│   └── components/        ← React компоненты
├── prisma/schema.prisma   ← Модель данных
└── .env                   ← Переменные окружения
```
