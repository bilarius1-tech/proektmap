## 01. Почему существует этот проект

Мир разработки изменился. Код перестал быть дефицитом. Любая AI-модель генерирует сайты, ботов, CRM за минуты.

Но количество неправильных инженерных решений не уменьшилось.

Большинство пользователей не знают:
- с чего начать проект
- какие этапы нельзя пропускать
- когда PostgreSQL, а когда SQLite
- зачем нужна Design System
- почему сначала данные, потом код

Современные AI-инструменты отлично отвечают на вопросы. Но они не сопровождают человека на протяжении всего жизненного цикла проекта.

**ProektMap существует чтобы превратить хаос разработки в понятную последовательность инженерных решений.**

---

## 02. Главная проблема рынка

Между генераторами кода и образовательными платформами — пустота.

| С одной стороны | С другой стороны |
|----------------|-----------------|
| AI пишет код за минуты | Курсы учат годами |
| Но не несёт ответственности за решения | Но не помогают строить реальный проект |

Пользователь мечется: AI → Google → YouTube → GitHub → Документация → Форум → Снова AI.

**Главная проблема — отсутствие инженерного маршрута.**

## 03. Философия проекта

### Мы не заменяем разработчика. Мы помогаем им стать.

```
Человек + AI + ProektMap = инженерное мышление
```

### Три столпа

1. **Последовательность важнее скорости.** Лучше пройти 4 фазы правильно, чем написать код за час и переписывать месяц.

2. **Понимание важнее копирования.** Каждый выбор объясняется: ПОЧЕМУ этот инструмент, а не другой.

3. **Инженерное мышление — главная ценность.** Код можно переписать. Интерфейс переделать. Стек заменить. Мышление остаётся навсегда.

### Конечная цель

Не создание сайта, приложения или SaaS. **Формирование инженерного мышления.**

Если после прохождения Blueprint пользователь самостоятельно принимает более качественные решения — проект выполнил задачу.

---

## 04. Принципы проектирования

1. **Каждая задача = карточка.** Atomic. Понятная. С AI-объяснением.
2. **XP за выполнение.** Прогресс должен быть виден.
3. **AI объясняет, не приказывает.** Пользователь принимает решение.
4. **Русский язык — основной.** Технические термины — на английском.
5. **Без сложного жаргона.** Объясняем как новичку.
6. **Модульность.** Каждый Blueprint — независимый модуль.
7. **Документация важнее кода.** Manifest > Architecture > Code.

---

## 05. Что такое Blueprint

Blueprint — это инженерный маршрут для конкретного типа проекта.

```
Blueprint "Сайт компании"
├── Фаза 1: Структура
│   ├── Карточка: Карта страниц
│   ├── Карточка: Design Tokens
│   └── Карточка: Структура данных
├── Фаза 2: Технологии
│   ├── Карточка: Выбор фреймворка
│   └── Карточка: База данных
├── Фаза 3: Реализация
│   ├── Карточка: Формы
│   └── Карточка: Telegram бот
└── Фаза 4: Запуск
    ├── Карточка: Домен и DNS
    └── Карточка: SSL и метрики
```

---

## 06. Decision Engine

Каждая карточка содержит точку принятия решения:

```
Вопрос: «Next.js или Vue?»
  ├── Контекст: ваш проект — сайт компании с блогом
  ├── AI-анализ: сравнивает варианты по 5 критериям
  ├── Рекомендация: Next.js (причина: экосистема, SEO, Vercel)
  └── Пользователь выбирает → решение сохраняется в историю
```

---

## 11. Новичок как главный пользователь

Каждое решение в интерфейсе проверяется вопросом: «Поймёт ли новичок?»

- Без профессионального жаргона
- Каждый термин объясняется при первом появлении
- AI доступен на каждом шаге
- Прогресс визуализирован (RPG-механика)

---

## 12. AI Architect

Внутренний AI-ассистент:

- Объясняет каждое решение
- Сравнивает альтернативы
- Проверяет выполненные задачи
- Предлагает улучшения

**Не заменяет пользователя. Помогает принимать решения.**

---

## 13. AI Curator

Отвечает за качество Blueprint'ов:
- Проверяет актуальность карточек
- Обновляет рекомендации при изменении технологий
- Предлагает новые Blueprint'ы

---

## 14. AI Reviewer

Проверяет решения пользователя:
- «Ты выбрал SQLite для high-load проекта. Уверен?»
- «Для интернет-магазина лучше PostgreSQL. Вот почему...»

---

## 15. MVP

**Что входит:**
- 1 Blueprint: «Сайт компании»
- 4 фазы, 12-15 карточек
- XP-система
- AI-консультант (OpenRouter)
- Email Magic Link

**Что НЕ входит:**
- Множественные Blueprint'ы
- Командная работа
- Собственные Blueprint'ы пользователей
- Векторный поиск
- Мобильное приложение

## 21. Видение через 5 лет

ProektMap стал стандартом инженерного сопровождения для русскоязычных разработчиков.

- 100+ Blueprint'ов для разных типов проектов
- Сообщество инженеров создаёт и проверяет Blueprint'ы
- AI-ассистент понимает контекст проекта и предлагает решения уровня senior-разработчика
- ProektMap используется в университетах и на курсах

**Главная метрика успеха:** количество людей, которые после ProektMap принимают инженерные решения лучше, чем до него.
